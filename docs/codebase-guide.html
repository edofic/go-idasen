02 · Bluetooth and desk protocol

BLE is services and characteristics, not a socket

TinyGo Bluetooth uses Linux BlueZ over D-Bus. The OS handles pairing; the program enables the existing adapter and connects to the configured MAC address.

Web analogy

WebBLE/GATT here
Discover an API’s endpointsDiscover services, then characteristics
GET /heightRead the height characteristic
POST /moveWrite a command/reference characteristic
WebSocket pushEnable characteristic notifications

Connect sequence

1Parse the MAC, call the adapter’s Connect in a goroutine, and let a context impose a timeout.
2Discover services/characteristics and cache them by UUID in a map.
3Send the Linak/IDÅSEN wake-up handshake, then use vendor UUIDs for commands and telemetry.

Explore the four-byte height packet

The response is two little-endian 16-bit numbers: an unsigned height offset and signed speed. Height is offset / 10000 + 0.62 metres. Speed is int16 / 10000 m/s.

height = uint16LE(bytes[0:2]) / 10000 + 0.62
speed  =  int16LE(bytes[2:4]) / 10000

// for 1.10 m:
offset = (1.10 - 0.62) × 10000 = 4800
bytes  = C0 12 … …    // little-endian

Safety boundary: EncodeHeight rejects any target outside 0.62–1.27 m before it can become a device write.

Commands

up     47 00  (without response)
down   46 00
stop   FF 00
wakeup FE 00
target = encoded height to reference UUID

Fast manual controls skip a reply. Target movement writes a desired height repeatedly.

MoveTo is a feedback loop

It wakes and stops the desk, writes the target every 100 ms, reads height/speed, and emits live updates. It finishes within 5 mm, or fails when stalled, timed out, or cancelled.

A deferred Stop() attempts to leave the physical desk stopped even if the command errors or Ctrl-C arrives.
03 · Persistent controller API

A local streaming API for a responsive UI

CLI/tray operations reconnect each time. idasen controller instead holds one BLE connection and streams state, avoiding reconnect latency for every widget click.

Protocol shape

// stdin commands
{"action":"move","position":"sit"}
{"action":"adjust","delta":0.005}
{"action":"set_position","position":"stand","height":1.1}
{"action":"stop"}

// stdout events
{"type":"status","status":"connecting"}
{"type":"height","status":"ready","height":0.75,"speed":0}

JSON Lines makes framing deliberately boring: QML’s SplitParser receives complete lines; Go’s scanner reads commands line-by-line.

Try the contract (simulation only)

Single event writer

A mutex surrounds JSON encoding, so simultaneous polling/movement/error goroutines cannot mix bytes into invalid stdout JSON.

One active move

A new target cancels the current movement and becomes pendingTarget; the latest request starts after the previous goroutine ends.

Idle polling

Every 250 ms, when not moving, it reads height. During movement, moveTo supplies 100 ms updates, avoiding competing reads.

04 · DankMaterialShell widget

QML is a client—not a Bluetooth implementation

The clean boundary: QML owns display and child-process I/O; Go owns hardware, validation, persistence, and cancellation. This is the same separation a web app keeps between frontend and backend.

Lifecycle

1Component.onCompleted starts idasen controller with stdin enabled.
2send() writes serialized JSON plus newline. QML helpers make button commands.
3stdout: SplitParser sends each line to handleEvent, which updates properties.
4Declarative QML bindings redraw. If the process exits, a 5-second timer restarts it unless shutting down.

Reactive event mapping

EventQML state/UI
heightSets deskHeight/deskSpeed; derived heightText rerenders.
positionsReplaces positions; Sit/Stand labels and editors update.
errorShows an error for six seconds with errorTimer.
process exitSets disconnected status and schedules recovery.
Unit translation belongs at the presentation edge: QML shows centimetres and millimetres, then sends metres (cm / 100, adjustmentMm / 1000). The controller protocol remains consistently SI.

06 · Packaging and tests

Deployment understands desktop reality

Nix packaging

flake.nix builds Linux packages, installs QML plugin files, and replaces @IDASEN_BINARY@ with the exact Nix-store executable. Home Manager links the plugin into DankMaterialShell’s config.

That binary substitution avoids relying on a desktop service’s often-minimal PATH.

Test the hardware-free seams

Tests cover packet encode/decode, bounds, command/config validation, and map cloning. Invalid MACs are rejected before touching an adapter.

BLE itself remains an integration concern, but the important protocol/control logic is testable without a desk.
